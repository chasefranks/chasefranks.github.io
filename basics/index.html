I'm a firm believer that as you learn new technology, you should start slow and focus on what you know first. You should occasionally go back to the command line to make sure you understand what is going on. Here are a collection of posts which I believe are in the spirit of learning from the ground up.

How JUnit Works

JUnit is one of the most often used tools in Java development, but do you really know how it works? Let's learn by stepping into the code and writing our own JUnit Runner!

Multithreading in Java

Using multiple threads in Java is easy! Here we break it down to the lowest level, to understand how the JVM creates and manages threads.

Write Your Own MOJO

We all probably run mvn clean package at least a 100 times a day, but do you really know what Maven is doing with your code? Let's dive in by writing our own Maven plugin!

Linux Namespaces

Let's talk about Linux namespaces. Did you know that namespaces are the basic building blocks of tools like Docker? Let's write some C code to create our own container runtime.

The Java Debugger

When you are debugging your code in Eclipse or IntelliJ, do you really understand what is happening at the JVM and with the bytecode?
